# Microcopy Guidelines

## Principles

- **Be helpful, not clever**: "Your password needs at least 8 characters" > "That's not going to work"
- **Be concise**: Cut every unnecessary word. Users don't read, they scan.
- **Be human**: Write like a helpful colleague, not a legal document.
- **Be specific**: "Something went wrong" is useless. "We couldn't save your changes. Please try again." is helpful.

## Common Patterns

| Context | Bad | Good |
|---------|-----|------|
| Error | "Error 500: Internal Server Error" | "Something went wrong. We've been notified and are working on it." |
| Empty state | "No results" | "No projects yet. Create your first project to get started." |
| Loading | "Loading..." | "Finding the best results for you..." |
| Success | "Form submitted" | "Your report has been sent. You'll receive a confirmation email shortly." |
| 404 | "Page not found" | "This page doesn't exist. It may have been moved or deleted." |

## Button Labels

- Use verbs: "Save changes" (not "Submit"), "Create account" (not "OK")
- Be specific about what happens: "Delete file permanently" (not just "Delete")
- Avoid "Yes/No" — use action labels: "Keep editing" / "Discard changes"

## Error Messages

Three parts:
1. **What happened** (in plain language)
2. **Why it happened** (if useful to the user)
3. **What to do next**

Example: "We couldn't save your changes because your internet connection was interrupted. Please check your connection and try again."

## Tone

- **B2B / Enterprise**: Professional, direct, respectful
- **B2C / Consumer**: Friendly, warm, encouraging
- **Developer tools**: Precise, technical, transparent
- **Medical / Finance**: Reassuring, compliant, clear
